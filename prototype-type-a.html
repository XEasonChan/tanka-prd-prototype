Type A: State Mutation Card

状态变更型 - 支持 HubSpot、Linear、Jira 等应用的状态更新

建议样例：

Identifying 状态

用户输入指令后，AI 正在识别意图和提取关键信息。

  • 显示 Skeleton Screen
  • AI 识别动作类型和目标对象
  • 查询 tool dictionary 获取 appId
触发条件
用户输入指令
1

意图识别

分析用户指令，识别动作类型

2

查询 Tool Dictionary

获取对应的 appId 和操作类型

3

检查连接状态

查询 integration status

Drafting 状态

意图识别完成，展示 Action Card 供用户确认。

  • Diff View 展示当前值与建议值
  • 用户可修改参数
  • 提供 Confirm / Cancel 按钮
  • Personal IM：收件人字段支持搜索联系人（按名称或电话号码）
触发条件
意图识别完成且 App 已连接
Personal IM 特性
• 点击 Edit 可搜索联系人
• 显示联系人名称 + 电话号码
• 支持手动编辑/修正联系人信息
1

获取当前值

从 API 读取当前状态

2

展示 Diff View

对比当前值和建议值

3

等待用户确认

Confirm 或 Cancel

Pending 状态

用户点击 Confirm 后，正在调用 API 执行更新。

  • UI 冻结，按钮显示 Spinner
  • 调用目标 API 进行更新
  • 支持 Optimistic UI
触发条件
用户点击 Confirm

Fulfilled 状态

API 返回成功，显示完成状态。

  • 显示绿勾和成功提示
  • 触发 Next Step 建议
  • 卡片变为静态展示
触发条件
API 返回 200 OK

Rejected 状态

API 返回错误，显示失败原因。

  • 显示红色警告和错误原因
  • 提供 Retry / Edit 按钮
  • 如果是未连接导致，触发 Suggest Link
触发条件
API 返回 4xx/5xx

Not Connected 状态

检测到目标 App 未连接，展示 Suggest Link Card。

  • 展示连接引导卡片
  • 说明为什么需要连接
  • 提供 Learn more / Connect 按钮
触发条件
integration status = false
