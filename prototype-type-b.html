Type B: Content Dispatch Card

内容分发型 - Send Email, Slack Message

建议样例：

Generating 状态

AI 正在根据用户指令生成邮件/消息草稿。

  • 显示 Skeleton Screen
  • AI 分析上下文生成内容
  • 提取收件人、主题等信息
触发条件
用户请求发送邮件/消息
1

分析用户指令

识别收件人、主题、内容要求

2

生成草稿内容

根据上下文生成邮件正文

3

展示预览

供用户确认或编辑

Preview 状态

草稿生成完成，展示邮件预览供用户确认。

  • 显示完整邮件内容
  • 支持编辑修改
  • 提供 Send / Edit 按钮
触发条件
草稿生成完成

Sending 状态

用户确认后，正在调用 API 发送邮件。

  • UI 冻结，显示发送中状态
  • 调用 Gmail/Outlook API
  • 等待发送结果
触发条件
用户点击 Send

Sent 状态

邮件发送成功，显示完成状态。

  • 显示绿勾和成功提示
  • 提供查看已发送邮件的入口
  • 可触发后续 Action 建议
触发条件
API 返回发送成功

Failed 状态

邮件发送失败，显示错误原因。

  • 显示红色警告和错误原因
  • 提供 Retry / Edit 按钮
  • 如果是未连接导致，触发 Suggest Link
触发条件
API 返回错误

Suggest Link 状态

检测到 Gmail 未连接，展示连接引导卡片。

  • A1 规则：发送失败且原因是未连接
  • 展示 Why（为什么需要连接）
  • 提供 Learn more / Connect 按钮
触发条件
integration status = false
